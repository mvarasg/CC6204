Transfer Learning¶

En este notebook resolveremos un problema real usando redes pre-entrenadas y transfer learning.

Patrones geométricos de Kunisch¶

En el ao 1998, Norbert Kunisch (un reconocido arquelogo clsico alemn) public su libro "Ornamente Geometrischer Vasen", en donde clasific diferentes patrones geomtricos de vasijas griegas que se encuentran en diferentes museos de Europa. Su trabajo fue muy importante para determinar tiempos y ubicaciones de muchos objetos esparcidos por todo el continente.

Su libro es un compendio de los diferentes patrones geomtricos presentes en vasijas y las anotaciones textuales de los patrones en diferentes idiomas.

El problema¶

En el museo de arqueologa Schloss Eggenberg quieren realizar un anlisis automtico de patrones en vasijas usando el compendio de Kunisch. Para esto nosotros realizamos primero un trabajo de escaneo del libro entero y de OCR para extraer tanto los patrones como las anotaciones textuales.

Como resultado tenemos 348 patrones, clasificados en seis clases:

  • Ornamentos circulares
  • Ornamentos triangulares
  • Ornamenros rectangulares
  • Pictografas
  • Trazos y lneas
  • Romboidales

Nuestro primer objetivo es encontrar una forma de lograr una alta eficacia de clasificacin de estos patrones. Sin embargo, al tener muy poca data, trataremos de resolverlo con la tcnica dle transfer learning.

  • Datos de train: 70%
  • Datos de val: 10%

    valNumber = math.ceil(0.1 * len(v))

    elemTrain = v[:trainNumber]

    assert (valNumber + testNumber + trainNumber) == len(v)

# Se utiliza funcin base del laboratorio 8, disponible en github.
def train_model(name, model, criterio, optimizer, scheduler, num_epochs = 25):
  since = time.time()

  best_model_wts = copy.deepcopy(model.state_dict())
  best_acc = 0.0
  train_losses = []
  val_losses = []
  train_accs = []
  val_accs = []
  print(f'Comenzando el entrenamiento del modelo {name}...')

  for epoch in range(num_epochs):
    print('Epoch {}/{}'.format(epoch+1, num_epochs))
    print('-' * 10)

    #Train model
    # scheduler.step()
    model.train()

    running_loss = 0.0
    running_corrects = 0.0

    for inputs, labels in train_loader:
        inputs = inputs.to(device)
        labels = labels.to(device)

        optimizer.zero_grad()

        outputs = model(inputs)
        _, preds = torch.max(outputs, 1)
        loss = criterion(outputs, labels)

        loss.backward()
        optimizer.step()
        scheduler.step()

        running_loss += loss.item() * inputs.size(0)
        running_corrects += torch.sum(preds == labels.data)

    epoch_loss_train = running_loss /len(train_dataset)
    epoch_acc_train = running_corrects.double() / len(train_dataset)
    train_losses.append(epoch_loss_train)
    train_accs.append(epoch_acc_train)

    print('Train Loss: {:.4f} Acc: {:.4f}'.format(epoch_loss_train, epoch_acc_train))

    #Validation
    model.eval()
    running_loss = 0.0
    running_corrects = 0.0

    for inputs, labels in val_loader:
      inputs = inputs.to(device)
      labels = labels.to(device)

      with torch.set_grad_enabled(False):
        outputs = model(inputs)
        _, preds = torch.max(outputs, 1)
        loss = criterion(outputs, labels)

      running_loss += loss.item() * inputs.size(0)
      running_corrects += torch.sum(preds == labels.data)

    epoch_loss_val = running_loss /len(val_dataset)
    epoch_acc_val = running_corrects.double() / len(val_dataset)
    val_losses.append(epoch_loss_val)
    val_accs.append(epoch_acc_val)

    print('Val Loss: {:.4f} Acc: {:.4f}'.format(epoch_loss_val, epoch_acc_val))

    if abs(epoch_acc_val-epoch_acc_train) <= 0.05:
      # print(abs(epoch_acc_val-epoch_acc_train))
      if epoch_acc_val > best_acc:
          best_acc = epoch_acc_val
          best_model_wts = copy.deepcopy(model.state_dict())

  time_elapsed = time.time() - since
  print('Training complete in {:.0f}m {:.0f}s'.format(time_elapsed//60, time_elapsed % 60))
  print('Best val accucary: {:.4f}'.format(best_acc))

  model.load_state_dict(best_model_wts)
  return [model, train_losses, val_losses, train_accs, val_accs, name, num_epochs]

class Modelo_custom(nn.Module):
    def __init__(self, modelo_base, n_clases, dropout_prob = 0.5, freeze_model = False):
        super(Modelo_custom, self).__init__()
        self.modelo_base = modelo_base

        if freeze_model:
            for param in self.modelo_base.parameters():
                param.requires_grad = False

        self.fc1 = nn.Linear(modelo_base.fc.out_features, 100)
        self.relu1 = nn.ReLU()
        self.dropout1 = nn.Dropout(p=dropout_prob)
        self.fc2 = nn.Linear(100, n_clases)
        # self.relu2 = nn.ReLU()
        # self.dropout2 = nn.Dropout(p=dropout_prob)
        # self.fc3 = nn.Linear(200, n_clases)

    def forward(self, x):
        x = self.modelo_base(x)
        x = self.fc1(x)
        x = self.relu1(x)
        x = self.dropout1(x)
        x = self.fc2(x)
        # x = self.relu2(x)
        # x = self.dropout2(x)
        # x = self.fc3(x)
        return x

Respuesta:¶

El problema debe ser afrontado con Fine Tunning, dado que entrenar directamente con la data disponible no dara buenos resultados al tener tan pocos datos. Se utiliza la CNN pre-entrenada Resnet18 y se agregan capas posteriores para realizar la clasificacin. Notar que la red CNN obtiene las caractersticas de las imagenes y mediante Fine Tunning, se re-ajustan los pesos de toda la nueva arquitectura.

En un comienzo se prob con la red ResNet18 con la misma configuracin de las capas fc, incluyendo la capa no lineal ReLu(). Dada la cantidad de la data disponible para entrenar fue necesario realizar data Augmentation al dataset de train. Esta consisti en realizar una serie de transformaciones a la data de train, como pueden ser rotaciones y flips. Despus de mucha prueba de error se decidi dejar al arquitectura y la data de train como muestra el presente notebook.

Los mejores resultados obtenidos muestran que en el conjunto de validacin se obtiene un accuracy de un 96% y un 91% en el conjunto de entrenamiento. Al probar en el conjunto de test, se obtiene un accuracy de un 0.96. Estos resultados son muy buenos, mostrando que la arquitectura aprende y, ms an, logra generalizar.

Agregar Dropout en clasificacin fue clave para reducir el sobreajuste, por lo que se estima que fue una muy buena estrategia.

Se evidencia que un problema como este requiere de prueba y error. Lo que ms podra acelerar el proceso es contar con ayuda de un experto del tema a tratar, conocer ms sobre el dataset y con esto hacer un buen pre-procesado de datos. Por ejemplo, despus de observar un poco las imagenes se lleg a la conlusin que algunas se parecen mucho a otras rotadas, por lo que se aplic esa transformacin al conjunto de entrenamiento. Data Augmentation fue fundamental, dado que permiti agregar la variabilidad necesaria para aprender.

La matriz de confusin muestra que el resultado general es excelente, practicamente clasifica bien todas las clases.

Nota: al no estar completamente seguro de si la semilla bastar para replicar, se adjunta el modelo.
